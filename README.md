# FLOW WMS — EC特化型 SaaS 倉庫管理システム

## プロダクトコンセプト

**FLOW WMS** は、EC事業者・3PL事業者向けに設計したSaaS型倉庫管理システム（WMS）です。

月間受注 1,000〜50,000 件規模の自社倉庫EC・D2C・中小3PLが、Amazon/楽天/Shopifyとの自動連携・ヤマト/佐川/ゆうパック送り状の自動発行を含む本格的な倉庫管理を、月額 ¥30,000〜 から利用できます。

某企業が5,000万〜2億円・6〜18ヶ月かけて実現する機能を、SaaSで1/10のコスト・2週間以内に提供することが最大の価値提案です。

## 主な機能

| 機能 | 概要 |
|------|------|
| **EC受注連携** | Amazon/楽天/Yahoo!/Shopify/BASE 受注の自動取込・在庫同期 |
| **送り状自動発行** | ヤマト運輸/佐川急便/日本郵便 と API連携し送り状を自動印刷 |
| **返品管理** | EC返品の検品・在庫戻し・Shopify/Amazon 返金API連携 |
| **3PL対応** | マルチクライアント管理・クライアントポータル・請求明細自動生成 |
| 入荷管理 | 発注照合・検品・入庫登録・ラベル発行 |
| 出荷管理 | ピッキング指示（4方式）・梱包・出荷確定 |
| 在庫管理 | リアルタイム在庫照会・棚卸・ロット/シリアル管理 |
| ロケーション管理 | 棚番管理・ゾーニング・空き容量監視 |
| KPI ダッシュボード | 入出荷件数・作業効率・誤出荷率・リードタイム可視化 |

### 技術スタック

- **フレームワーク**: Next.js 15 (App Router)
- **言語**: TypeScript
- **データベース**: PostgreSQL
- **キャッシュ**: Redis
- **スタイリング**: Tailwind CSS
- **ORM**: Prisma（予定）
- **コンテナ**: Docker / Docker Compose

## ディレクトリ構成

```
wms/
├── specs/      # 仕様書・設計ドキュメント（画面仕様・DB設計・API定義など）
├── db/         # データベース関連（マイグレーション・シード・ERD）
└── src/        # アプリケーションソースコード（Next.js プロジェクト本体）
```

### 各ディレクトリの役割

- **`specs/`** — 機能仕様書、画面設計書、API 仕様書、DB 論理設計書を管理します。実装前に仕様を固めるドキュメント置き場です。詳細は [specs/README.md](./specs/README.md) を参照してください。
- **`db/`** — PostgreSQL のマイグレーションファイル、初期データ投入用シードスクリプト、ER 図などを管理します。
- **`src/`** — Next.js アプリ本体。`app/`（ルーティング）、`components/`（UI）、`lib/`（ユーティリティ）、`server/`（API ロジック）などで構成されます。

## セットアップ（将来の実装時用テンプレート）

### 1. 前提条件

- Node.js 20 以上
- Docker / Docker Compose
- Git

### 2. 依存パッケージのインストール

```bash
npm install
```

### 3. 環境変数の設定

```bash
cp .env.example .env.local
# .env.local を編集して DB 接続情報などを設定してください
```

### 4. Docker で DB を起動

```bash
docker compose up -d
```

### 5. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開いてください。
