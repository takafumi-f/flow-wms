## 1. 入荷管理（Receiving）

### F-01 入荷予定登録

**概要**: 入荷予定情報を登録し、入荷検品・ロケーション割付の基準データを作成する。

| 要件ID | 要件内容 |
|--------|---------|
| F-01-01 | ERP・発注管理システムから受信した発注データを自動取込み、入荷予定として登録できること |
| F-01-02 | 発注データとの自動連携に加え、手動での入荷予定入力が可能であること |
| F-01-03 | 入荷予定登録時に、品目コード・品目名・数量・予定入荷日・仕入先・ロット番号・賞味期限（任意）を登録できること |
| F-01-04 | 入荷予定の一覧表示・検索（仕入先・品目・日付範囲）ができること |
| F-01-05 | 入荷予定を修正・キャンセルできること（入荷実績登録後はキャンセル不可） |
| F-01-06 | 入荷予定をCSV形式で一括インポートできること |

### F-02 入荷検品

**概要**: ハンディターミナルまたはPC画面を用いて、入荷した商品の検品を行う。

| 要件ID | 要件内容 |
|--------|---------|
| F-02-01 | バーコード（JAN/EAN/ITF）またはQRコードのスキャンにより品目を特定できること |
| F-02-02 | スキャン後、入荷予定データと照合し、品目・ロット番号・数量の一致確認ができること |
| F-02-03 | 賞味期限・製造年月日のスキャンまたは手動入力ができること |
| F-02-04 | スキャン結果が入荷予定と不一致の場合、警告を表示し確認を促すこと |
| F-02-05 | 検品数量を都度入力または1スキャン1個加算方式で選択できること |
| F-02-06 | ハンディターミナルでの検品時、スキャン応答時間は500ms以内であること |
| F-02-07 | 入荷検品作業をハンディターミナルとPC画面の両方から実施できること |

### F-03 入荷ロケーション自動割付

**概要**: 検品済み商品を保管するロケーションを自動的に決定する。

| 要件ID | 要件内容 |
|--------|---------|
| F-03-01 | 入荷検品完了後、空きロケーションを自動検索してロケーションを割付けること |
| F-03-02 | ロケーション割付の優先条件を設定できること（同一品目の隣接保管、温度帯一致、重量制限遵守） |
| F-03-03 | 自動割付されたロケーションを作業者が変更できること |
| F-03-04 | ロケーションの空き状況をリアルタイムで参照し、最新情報に基づき割付を行うこと |
| F-03-05 | 割付結果をハンディターミナルに配信し、格納ガイドを表示できること |

### F-04 入荷実績登録・差異処理

**概要**: 検品完了後の入荷実績を確定し、入荷予定との差異を処理する。

| 要件ID | 要件内容 |
|--------|---------|
| F-04-01 | 入荷検品完了後、入荷実績として確定登録できること |
| F-04-02 | 入荷確定と同時に在庫がリアルタイムに更新されること |
| F-04-03 | 入荷予定数量と実績数量の差異を自動検出し、差異一覧を表示できること |
| F-04-04 | 差異の処理方法（予定数量の修正・追加入荷扱い・返品扱い）を選択できること |
| F-04-05 | 差異処理の内容を記録し、差異処理履歴として参照できること |
| F-04-06 | 入荷実績に基づく入荷確認書（受領書）をPDF形式で発行できること |

### F-05 入荷履歴照会

**概要**: 過去の入荷実績を検索・参照できる。

| 要件ID | 要件内容 |
|--------|---------|
| F-05-01 | 仕入先別・品目別・日付範囲・ロット番号での入荷履歴検索ができること |
| F-05-02 | 入荷履歴の詳細（品目・数量・ロット・賞味期限・担当者・割付ロケーション）を参照できること |
| F-05-03 | 入荷履歴をCSV形式でエクスポートできること |

## 2. 出荷管理（Shipping）

### F-06 出荷指示

**概要**: 受注データに基づき出荷指示を生成する。

| 要件ID | 要件内容 |
|--------|---------|
| F-06-01 | ERP・受注管理システムから受信した受注データを自動取込み、出荷指示を生成できること |
| F-06-02 | 受注データとの自動連携に加え、手動での出荷指示作成が可能であること |
| F-06-03 | 出荷指示には出荷先・品目・数量・出荷予定日・配送方法・優先度を含めること |
| F-06-04 | 在庫不足の場合、出荷指示作成時に警告を表示し、部分出荷または保留の選択ができること |
| F-06-05 | 出荷指示の一覧表示・ステータス管理（未ピッキング・ピッキング中・検品待ち・出荷済み）ができること |
| F-06-06 | 出荷指示のキャンセル・修正ができること（ピッキング開始後は権限者のみ可能） |

### F-07 ピッキング指示（4方式対応）

**概要**: 出荷指示に基づき、作業者へピッキング指示を配信する。

| 要件ID | 要件内容 |
|--------|---------|
| F-07-01 | **シングルピッキング**: 1出荷指示に対して1人の作業者が担当するピッキング方式を選択できること |
| F-07-02 | **バッチピッキング**: 複数の出荷指示をまとめて1人の作業者がピッキングする方式を選択できること（最大50件/バッチ） |
| F-07-03 | **ゾーンピッキング**: 倉庫を複数のゾーンに分け、ゾーン担当者がそれぞれピッキングする方式を選択できること |
| F-07-04 | **ウェーブピッキング**: 時間帯・優先度・出荷便に基づき出荷指示を波として束ね、一括ピッキングする方式を選択できること |
| F-07-05 | ピッキング方式は出荷指示ごとまたは一括で選択できること |
| F-07-06 | ピッキング順序は最短経路（ロケーションの並び順）で自動計算すること |
| F-07-07 | 在庫引当方式（FIFO・FEFO・指定ロット）をシステム設定で選択できること |

### F-08 ピッキングリスト発行・ハンディ端末配信

**概要**: ピッキング指示をピッキングリストまたはハンディ端末画面として提供する。

| 要件ID | 要件内容 |
|--------|---------|
| F-08-01 | ピッキングリストをPDF形式で印刷発行できること |
| F-08-02 | ピッキング指示をハンディターミナルへリアルタイム配信できること |
| F-08-03 | ハンディターミナル画面に対象ロケーション・品目・数量・バーコードを表示できること |
| F-08-04 | 作業者がピッキング完了をスキャンにより報告できること（ロケーション・品目・数量の順でスキャン確認） |
| F-08-05 | 誤ロケーション・誤品目のスキャン時にアラームと警告表示を行うこと |
| F-08-06 | ピッキング中に品切れが発生した場合、代替ロケーションまたは短納品の選択肢を提示すること |

### F-09 検品・梱包指示

**概要**: ピッキング完了品の二次検品と梱包作業を管理する。

| 要件ID | 要件内容 |
|--------|---------|
| F-09-01 | ピッキング完了後、出荷検品画面でスキャンによる全数検品ができること |
| F-09-02 | 出荷先・品目・数量・ロット番号の照合チェックを自動実施すること |
| F-09-03 | 梱包方法（個口数・梱包サイズ・特記事項）を登録できること |
| F-09-04 | 梱包完了後、出荷ラベル（出荷先・品目・バーコード）をラベルプリンタへ送信できること |
| F-09-05 | 梱包作業の進捗をリアルタイムで管理画面に反映すること |

### F-10 出荷実績登録・出荷確認書発行

**概要**: 出荷完了の記録と証憑書類の発行を行う。

| 要件ID | 要件内容 |
|--------|---------|
| F-10-01 | 出荷完了時に出荷実績を確定登録できること |
| F-10-02 | 出荷確定と同時に在庫がリアルタイムに減算されること |
| F-10-03 | 出荷確認書（納品書・送り状）をPDF形式で発行できること |
| F-10-04 | 出荷確認書のフォーマットを出荷先ごとにカスタマイズできること |
| F-10-05 | 出荷実績データをERP・EDIへ自動送信できること |

### F-11 出荷進捗リアルタイム表示

**概要**: 出荷オペレーション全体の進捗状況をリアルタイムで可視化する。

| 要件ID | 要件内容 |
|--------|---------|
| F-11-01 | 出荷指示ごとのステータス（未着手・ピッキング中・検品待ち・梱包中・出荷済み）をリアルタイム表示すること |
| F-11-02 | 作業者ごとのピッキング進捗・完了件数をリアルタイム表示すること |
| F-11-03 | 出荷便別の積み込み状況・出荷完了率を表示すること |
| F-11-04 | 遅延リスクがある出荷指示（制限時間の 70%経過）をハイライト表示すること |
| F-11-05 | WebSocketによるプッシュ通知で画面リロードなしに進捗が更新されること |

## 3. 在庫管理（Inventory）

### F-12 リアルタイム在庫照会

**概要**: 現在の在庫状況を様々な軸で照会できる。

| 要件ID | 要件内容 |
|--------|---------|
| F-12-01 | ロケーション別・品目別・ロット別・賞味期限別の在庫照会ができること |
| F-12-02 | 在庫照会画面は3秒以内に結果を返すこと |
| F-12-03 | 在庫状況（有効在庫・引当済み・フリー在庫）を区分して表示すること |
| F-12-04 | バーコードスキャンによる品目特定・在庫照会ができること（ハンディ端末対応） |
| F-12-05 | 品目の保管ロケーション一覧（複数ロケーション保管対応）を表示できること |
| F-12-06 | 在庫照会結果をCSV形式でエクスポートできること |

### F-13 在庫引当

**概要**: 出荷指示に対し、適切な在庫を引当てる。

| 要件ID | 要件内容 |
|--------|---------|
| F-13-01 | 在庫引当方式としてFIFO（先入先出）・FEFO（期限優先）・指定ロットを選択できること |
| F-13-02 | 引当方式は品目マスタまたは出荷指示単位で設定できること |
| F-13-03 | 引当済み在庫は他の出荷指示から重複引当されないよう排他制御すること |
| F-13-04 | 引当のキャンセル・変更が権限者により実施できること |
| F-13-05 | 同一品目が複数ロケーションにある場合、引当最適化（ロケーション集約・端数処理）を行うこと |

### F-14 在庫移動

**概要**: ロケーション間での在庫の移動を管理する。

| 要件ID | 要件内容 |
|--------|---------|
| F-14-01 | 移動元ロケーション・移動先ロケーション・品目・数量を指定した在庫移動指示を作成できること |
| F-14-02 | ハンディターミナルから移動指示を受け取り、スキャンにより移動完了を確認できること |
| F-14-03 | 在庫移動完了と同時に在庫ロケーション情報がリアルタイムに更新されること |
| F-14-04 | 在庫移動の履歴を記録し、トレーサビリティを確保すること |

### F-15 棚卸

**概要**: 実在庫と帳簿在庫の差異を確認・修正する棚卸機能。

| 要件ID | 要件内容 |
|--------|---------|
| F-15-01 | 全数棚卸（全ロケーションを一度に棚卸）を実施できること |
| F-15-02 | サイクルカウント（エリア・品目カテゴリ単位の部分棚卸）を通常業務と並行して実施できること |
| F-15-03 | 棚卸指示をハンディターミナルへ配信し、スキャンによる実地カウントができること |
| F-15-04 | 棚卸入力後、帳簿在庫との差異を自動計算して差異一覧を表示すること |
| F-15-05 | 差異確認後、在庫修正（棚卸確定）ができること（権限者承認フロー対応） |
| F-15-06 | 棚卸結果をレポート形式でPDF/CSVエクスポートできること |

### F-16 在庫アラート

**概要**: 在庫状況の異常を自動検知してアラートを発報する。

| 要件ID | 要件内容 |
|--------|---------|
| F-16-01 | 品目マスタに設定した安全在庫数量を下回った場合にアラートを発報すること |
| F-16-02 | 賞味期限・使用期限が設定した日数以内に迫った在庫にアラートを表示すること（デフォルト: 30日前） |
| F-16-03 | アラートはKPIダッシュボードおよびメール通知で確認できること |
| F-16-04 | アラート閾値は品目マスタ単位で設定できること |
| F-16-05 | アラートの確認・対応済みマークをユーザーが記録できること |

### F-17 在庫推移グラフ

**概要**: 品目別の在庫数量の推移をグラフで可視化する。

| 要件ID | 要件内容 |
|--------|---------|
| F-17-01 | 品目を指定して在庫数量の日次推移グラフを表示できること |
| F-17-02 | 表示期間（1週間・1ヶ月・3ヶ月・カスタム）を選択できること |
| F-17-03 | 入荷・出荷・調整の内訳を重ね表示できること |
| F-17-04 | 複数品目を同一グラフに重ねて比較表示できること |

## 4. ロケーション管理（Location）

### F-18 倉庫レイアウト定義

**概要**: 倉庫の物理的な構造をシステム上に定義する。

| 要件ID | 要件内容 |
|--------|---------|
| F-18-01 | 倉庫を「棟 → 階 → エリア → 棚 → 段 → 列」の階層構造で定義できること |
| F-18-02 | 各レベルの追加・編集・削除（在庫のないロケーションのみ）ができること |
| F-18-03 | 倉庫レイアウトを平面図（テキストベースのグリッド表示）で確認できること |
| F-18-04 | ロケーションコードの採番ルールを設定できること（例: A-01-03-02 = Aエリア-棚01-段03-列02） |

### F-19 ロケーション属性管理

**概要**: 各ロケーションの保管条件・制約を管理する。

| 要件ID | 要件内容 |
|--------|---------|
| F-19-01 | ロケーションに温度帯（常温・冷蔵・冷凍）を設定できること |
| F-19-02 | ロケーションに最大積載重量を設定できること |
| F-19-03 | ロケーションに保管可能な品目カテゴリ制限を設定できること |
| F-19-04 | ロケーションの有効・無効ステータスを管理できること（メンテナンス中等） |
| F-19-05 | 品目の保管条件（温度帯・重量）とロケーション属性を照合し、不適切な割付を防止すること |

### F-20 ロケーション稼働率可視化

**概要**: 倉庫全体・エリア別のロケーション利用状況を可視化する。

| 要件ID | 要件内容 |
|--------|---------|
| F-20-01 | 倉庫全体・エリア別のロケーション稼働率（使用中/全体）をダッシュボードに表示すること |
| F-20-02 | ロケーション稼働率をヒートマップ形式でグリッド表示できること |
| F-20-03 | 長期間移動のない在庫（デッドストック）が存在するロケーションを強調表示できること |

### F-21 棚割最適化提案

**概要**: 出荷頻度・品目特性に基づく最適な棚割を提案する。

| 要件ID | 要件内容 |
|--------|---------|
| F-21-01 | 過去の出荷データを分析し、高頻度出荷品目を出荷口近くのロケーションへ移動する提案を生成できること |
| F-21-02 | 棚割最適化提案を一覧表示し、適用する提案を選択できること |
| F-21-03 | 提案を適用すると在庫移動指示（F-14）が自動生成されること |

## 5. 作業指示・進捗管理（Task Management）

### F-22 作業者への指示配信

**概要**: 作業者のハンディターミナルへ作業指示を配信する。

| 要件ID | 要件内容 |
|--------|---------|
| F-22-01 | 作業者ごとにピッキング・格納・在庫移動・棚卸の指示をハンディターミナルへ配信できること |
| F-22-02 | 指示の優先度・緊急度を設定でき、ハンディ画面に表示できること |
| F-22-03 | 指示の完了・中断・スキップを作業者がハンディ端末から報告できること |
| F-22-04 | 作業者が複数の指示を受け持つ場合、効率的な順序でナビゲーションすること |

### F-23 作業進捗リアルタイム監視

**概要**: 管理者が全作業者の進捗をリアルタイムで把握できる。

| 要件ID | 要件内容 |
|--------|---------|
| F-23-01 | 管理画面で全作業者の現在の作業内容・進捗率をリアルタイム表示すること |
| F-23-02 | 作業指示別の完了率・残作業量を表示すること |
| F-23-03 | 指示なし（待機中）の作業者をリスト表示し、追加指示の割り振りができること |
| F-23-04 | WebSocketにより進捗更新が即座に画面反映されること（ポーリング不要） |

### F-24 作業実績・生産性レポート

**概要**: 作業者・チームの生産性をレポート形式で提供する。

| 要件ID | 要件内容 |
|--------|---------|
| F-24-01 | 作業者別の1時間あたりピッキング件数・処理数量を集計できること |
| F-24-02 | 作業者別のエラー率（スキャンミス・ピッキングミス）を集計できること |
| F-24-03 | 日次・週次・月次のレポートを自動生成し、ダッシュボードに表示できること |
| F-24-04 | レポートをPDF/CSVでエクスポートできること |

### F-25 フォークリフト・AGV作業指示連携

**概要**: フォークリフト端末およびAGVへの作業指示連携を行う。

| 要件ID | 要件内容 |
|--------|---------|
| F-25-01 | フォークリフト端末（タブレット）へパレット搬送指示を配信できること |
| F-25-02 | AGV連携APIを通じてAGVへ搬送指示を送信できること（REST API/MQTTに対応） |
| F-25-03 | AGVの稼働状況・位置情報をフィードバック受信し、管理画面で表示できること |
| F-25-04 | AGV障害発生時に手動フォークリフト対応へ切り替えられること |

## 6. マスタ管理（Master Data）

### F-26 品目マスタ

**概要**: 取り扱う全品目の基本情報を管理する。

| 要件ID | 要件内容 |
|--------|---------|

## 10. 3PL（物流代行）マルチクライアント対応

### F-44 マルチクライアント在庫管理

**概要**: 1つのFLOW WMSインスタンスで複数の荷主企業（クライアント）の在庫を完全独立管理する。

| 要件ID | 要件内容 |
|--------|---------|
| F-44-01 | クライアント（荷主企業）を複数登録し、それぞれ独立した在庫空間として管理できること |
| F-44-02 | PostgreSQL RLSによりクライアントごとのデータを完全分離し、他クライアントデータへのアクセスを物理的に不可とすること |
| F-44-03 | ロケーションをクライアントごとに割り当て、保管エリアを物理・論理で分離管理できること |
| F-44-04 | クライアント別の入出荷実績・在庫サマリー・KPIを月次レポートとして自動生成できること |
| F-44-05 | クライアントの追加・停止・データ移行をシステム管理者（3PL事業者）が実施できること |

### F-45 クライアントポータル

**概要**: 荷主企業が自社在庫・入出荷状況を閲覧できる専用ポータルを提供する。

| 要件ID | 要件内容 |
|--------|---------|
| F-45-01 | クライアント専用ログイン（権限: 自社データ閲覧のみ）を提供できること |
| F-45-02 | クライアントは自社在庫照会・入出荷実績・月次レポートをポータルから取得できること |
| F-45-03 | クライアントポータルを白ラベル（クライアントロゴ・コーポレートカラー設定）で提供できること |
| F-45-04 | クライアントがポータル上で出荷依頼（出荷指示登録）を入力できること（3PL出荷依頼機能） |
| F-45-05 | クライアントポータルへのAPIアクセスを提供し、クライアントの社内システムから在庫照会できること |

### F-46 3PL請求管理

**概要**: クライアントごとの物流コストを自動集計し、月次請求明細を生成する。

| 要件ID | 要件内容 |
|--------|---------|
| F-46-01 | 保管料（パレット・棚段数単位）・入庫料・出荷料・荷役料・加工費をクライアント別に自動集計できること |
| F-46-02 | 月次請求明細書をPDF形式で自動生成し、クライアントポータルで配信できること |
