```
+-----------------------------------------------+
|                                               |
|           FLOW WMS                            |

|                                               |
|  +-------------------------------------------+|

|  |  [                                      ] ||
```

#### 入力項目

| 項目名 | 型 | 必須 | バリデーション |
|--------|-----|:----:|--------------|
| メールアドレス | text（email） | ○ | メール形式、最大255文字 |
| パスワード | password | ○ | 最低8文字 |

#### 操作

| 操作 | 処理 |
|------|------|
| ログインボタン押下 | API `/api/auth/login` を呼び出し。成功時は JWT をセッションに保存し `/` へリダイレクト |

#### 画面遷移

| 条件 | 遷移先 |
|------|--------|
| 認証成功 | SC-010 KPIダッシュボード |
| 既にログイン済み | SC-010 KPIダッシュボード |

#### バリデーションルール

- 5回連続認証失敗でアカウントを 15 分ロック
- ロック中は「アカウントがロックされています。15 分後に再試行してください」を表示
- パスワードは画面上にマスク表示。表示/非表示トグルボタンあり

### SC-021: 入荷予定登録・編集

- **URLパス**: `/receiving/new`, `/receiving/[id]/edit`
- **コンポーネント**: `app/(dashboard)/receiving/new/page.tsx`, `app/(dashboard)/receiving/[id]/edit/page.tsx`
- **目的**: 発注（PO）ヘッダおよび明細を登録・編集する
- **ハンディ端末対応**: なし

#### 画面レイアウト

```
+--------------------------------------------------------------+
| 入荷管理 > 入荷予定登録                                        |
+--------------------------------------------------------------+
| ■ 基本情報                                                    |
| 発注番号 * [            ]   取引先 * [              ▼]        |
| 予定入荷日 * [          ]   備考   [                        ] |
+--------------------------------------------------------------+
| ■ 発注明細                                          [+ 行追加]|
| # | 品目名（SKU）           | 発注数量 | 単価    | 小計      |
|---|------------------------|----------|---------|-----------|
| 1 | [品目検索...          ] | [     ]  | [     ] | ¥0        |
| 2 | [品目検索...          ] | [     ]  | [     ] | ¥0        |
|                                              合計: ¥0        |
+--------------------------------------------------------------+
| [キャンセル]                              [下書き保存] [登録]  |
+--------------------------------------------------------------+
```

#### 入力項目

| 項目 | 型 | 必須 | バリデーション |
|------|-----|:----:|--------------|
| 発注番号 | text | ○ | 最大50文字、英数字・ハイフン・アンダーバーのみ、一意 |
| 取引先 | select | ○ | 仕入先（supplier）のみ選択可 |
| 予定入荷日 | date | ○ | 当日以降の日付 |
| 備考 | textarea | - | 最大500文字 |

**明細行**

| 項目 | 型 | 必須 | バリデーション |
|------|-----|:----:|--------------|
| 品目 | 検索セレクト | ○ | 品目マスタに存在する品目 |
| 発注数量 | number | ○ | 1以上の整数 |
| 単価 | number | - | 0以上の数値 |

#### バリデーションルール

- 明細行は最低1行必要
- 同一品目は明細に重複登録不可
- 発注番号は既存POと重複不可（非同期チェック）

#### 操作

| 操作 | 処理 |
|------|------|
| 行追加 | 明細行を1行追加 |
| 行削除 | 対象明細行を削除（確認なし） |
| 下書き保存 | planned 状態で保存 |
| 登録 | 入力値バリデーション後に API 登録 |

#### 表示項目

| 項目 | 内容 |

| ロット番号 | 手動入力 |
| 賞味期限 | 日付入力 |
| 格納先ロケーション | 推奨ロケーションをデフォルト表示、変更可 |
| 検品数量 | 今回の格納数量 |
| 検品進捗一覧 | 各品目の進捗状況 |

#### バリデーションルール

- スキャンしたバーコードが PO の品目に含まれない場合はエラー表示
- 検品数量が予定数量を超える場合は確認ダイアログを表示（過多入荷）
- 格納先ロケーションの重量・容量制限チェック（API経由）
- 格納先ロケーションの温度帯と品目の保管条件が一致しない場合は警告

#### 操作

| 操作 | 処理 |
|------|------|
| バーコードスキャン | 品目情報を自動補完 |
| 格納確定 | `receiving_records` に記録し、`inventory` を更新 |
| 入荷完了ボタン | セッションを completed に更新（全品目完了時のみ押下可） |

### エラーハンドリング方針

| エラー種別 | 表示方法 | ユーザーアクション |
|-----------|---------|------------------|
| 入力バリデーションエラー | フィールド直下にインラインメッセージ | 入力修正 |
| 業務ルール違反 | 赤色トースト通知（5秒） | 内容確認・修正 |
| API エラー（4xx） | モーダルダイアログ | 再試行または前画面に戻る |
| API エラー（5xx） | モーダルダイアログ（管理者連絡案内） | 再試行 |
| 楽観的ロック競合 | 「他のユーザーが更新しました。最新データを読み込んでください」 | 再読み込み |
| ネットワーク切断 | 上部固定バナー「接続が切れました」 | 接続復旧を待つ |

### ハンディ端末対応方針

| 項目 | 内容 |
|------|------|
| 対象解像度 | 360px〜480px 幅 |
| フォントサイズ | 最小 16px（タップターゲット最小 44px） |
| ボタンサイズ | 高さ 56px 以上（手袋着用を考慮） |
| バーコードスキャン | カメラ API（`getUserMedia`） + USB HID 入力（キーボードエミュレーション） |
| オフライン対応 | Service Worker によるキャッシュ。スキャンデータをローカルキューに保存し、接続回復時に同期 |
| 画面輝度 | ダーク/ライトモード切替（倉庫内の明るさに応じて） |
